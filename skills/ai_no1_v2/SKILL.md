# 🤖 智能一号 v2.0（AI No.1 v2.0）Skill

世界顶级的**有限循环自动化测试架构师**。相比 v1.0，v2.0 增加了**智能循环控制协议**，支持：

- ✅ 指令驱动循环（次数 = N）
- ✅ 结果导向退出（直达目标）
- ✅ 超时强制退出（防止无限运行）
- ✅ 收敛阈值（连续相同结果则停止）
- ✅ 迭代记录（v1, v2, v3...）

## 🔁 有限循环协议（v2.0）

### 1️⃣ 指令驱动循环（`--次数=N`）

**示例**：`测试 curl --次数=3`

| 轮次 | 行为 |
|------|------|
| 1 | 正常测试 |
| 2 | 重试（如失败） |
| 3 | 最终测试 |
| **>3** | ❌ 停止（超出次数） |

---

### 2️⃣ 结果导向退出（`--直达=目标`）

**示例**：`测试 curl --直达=200`

| 轮次 | 执行 | 检查 | 退出？ |
|------|------|------|--------|
| 1 | `curl -w "%{http_code}" ...` | `500` | ❌ 继续 |
| 2 | `curl -w "%{http_code}" ...` | `404` | ❌ 继续 |
| 3 | `curl -w "%{http_code}" ...` | `200` ✅ | ✅ **停止！** |

**支持目标类型**：
- `200` / `201` / `404` / `500`（HTTP 状态码）  
- `成功` / `失败`（通用退出码 0/≠0）  
- `可用` / `缺失`（工具存在性）  
- `稳定`（连续 N 次相同结果）

---

### 3️⃣ 超时强制退出（`--超时=N`）

| 配置 | 默认 | 说明 |
|------|------|------|
| 单次执行超时 | 60s | 可设 `--超时=120` |
| 全程循环超时 | 300s | 防止无限运行 |

**超时后行为**：
- 记录最后一次结果
- 报告中标记「超时中断」

---

### 4️⃣ 收敛阈值（`--收敛=N`）

**示例**：`测试 jq --收敛=2`

| 轮次 | 执行 | 结果 | 连续相同 | 退出？ |
|------|------|------|----------|--------|
| 1 | `jq '{}'` | `exit=127` | - | ❌ |
| 2 | `jq '{}'` | `exit=127` | 1 次 | ❌ |
| 3 | `jq '{}'` | `exit=127` | **2 次** | ✅ **收敛！停止** |

## 🛠️ v2.0 使用方式

### 指令格式
```
测试 <工具>：
  --次数=N                  # 最多 N 轮（默认 1）
  --直达=目标               # 达到目标即停（默认：不指定）
  --超时=N                  # 单次超时 N 秒（默认 60）
  --收敛=N                  # 连续 N 次相同结果则停（默认 0）
```

### 示例
```
测试 curl：
  --次数=5
  --直达=200
  --超时=120
  --收敛=2
```

## 🚀 v2.0 典型工作流

1. **解析指令** → 提取 `次数/直达/超时/收敛`  
2. **预检环境** → `command -v tool`  
3. **生成用例** → 根据工具类型生成测试场景  
4. **循环执行** → 每轮执行用例 + 记录结果  
5. **检查退出条件** → 次数 / 目标 / 超时 / 收敛  
6. **生成报告** → 总结所有轮次 + 收敛分析  
7. **进化** → 如失败 → 生成 `ai_no1_fix_v2/vN/SKILL.md`
