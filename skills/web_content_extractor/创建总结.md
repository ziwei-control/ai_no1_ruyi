## 🎯 核心功能

### 1. 多策略提取引擎

| 方法 | 优先级 | 速度 | 成功率 | 适用场景 |
|------|--------|------|--------|----------|
| r.jina.ai API | 1️⃣ | ⚡⚡⚡ | ⭐⭐⭐⭐⭐ | 普通网页、博客、文档 |
| curl 移动端 | 2️⃣ | ⚡⚡ | ⭐⭐⭐⭐ | 微信公众号、知乎 |
| browser_use | 3️⃣ | ⚡ | ⭐⭐⭐ | 动态 JS 内容 |
| GitHub 搜索 | 4️⃣ | ⚡⚡ | ⭐⭐⭐ | 技术文章补充 |
| 缓存服务 | 5️⃣ | ⚡⚡ | ⭐⭐ | 备用方案 |

### 2. 智能 Fallback 机制

```
开始提取
   ↓
尝试 r.jina.ai
   ├─ 成功 → 提取 Markdown → 保存 ✅
   └─ 失败 → 下一步
   ↓
尝试 curl 移动端
   ├─ 成功 → 解析 HTML → 保存 ✅
   └─ 失败 → 下一步
   ↓
尝试 browser_use
   ├─ 成功 → 截图 + 快照 → 保存 ✅
   └─ 失败 → 下一步
   ↓
尝试缓存服务
   ├─ 成功 → 提取内容 → 保存 ✅
   └─ 失败 → 报告错误 ❌
```

### 3. 自动存档系统

**存档位置：** `/root/.copaw/memory/web_archive_<domain>_<timestamp>.md`

**档案内容：**
- ✅ 元数据（URL、时间、方法、状态）
- ✅ 提取的正文内容

- ✅ 建议和指导

## 🛡️ 安全和伦理

### 使用准则

- ✅ 仅用于个人学习和研究
- ✅ 尊重网站 robots.txt
- ✅ 控制请求频率（建议间隔 2-5 秒）
- ✅ 标注内容来源
- ❌ 不用于商业用途
- ❌ 不提取敏感个人信息
- ❌ 不大量爬取（每日<100 次）

### 最佳实践

```bash
# 批量提取时添加延迟
while read url; do
  ./extract.sh "$url"
  sleep 3  # 间隔 3 秒
done < urls.txt
```

## 📞 故障排除

### 常见问题

**Q: 所有方法都失败？**
A: 
1. 检查网络连接
2. 手动访问 URL 确认网站正常
3. 稍后重试
4. 建议用户提供截图或复制内容

**Q: 中文乱码？**
A:
```bash
# 手动转换编码
curl ... | iconv -f gb2312 -t utf-8
```

**Q: 存档文件过大？**
A:
1. 限制提取内容长度
2. 只提取前 N 个段落
3. 压缩存档文件

**Q: 脚本执行卡住？**
A:
1. 检查超时设置
2. 使用 `timeout` 命令包裹
3. 查看日志输出
