## 📋 v2.2 NLP 问答层设计

### 1️⃣ 意图识别（Intent Detection）

| 意图 | 关键词 | 说明 |
|------|--------|------|
| `test` | 测试, 测, 检查, 验证, 跑一下 | 主要意图 |
| `install` | 安装, 配置, add, install | 次意图（预留） |
| `fix` | 修复, 调试, 修正 | 次意图（预留） |

**识别逻辑**：
```python
if user_input matches [测试|测|检查] + [项目|脚本|程序|docker]:
    intent = "test"
elif user_input matches [安装|配置]:
    intent = "install"
else:
    intent = "unknown"
```

---

### 2️⃣ 实体抽取（Entity Extraction）

| 实体 | 关键词模式 | 说明 |
|------|------------|------|
| `path` | `/usr/...`, `./app.py`, `/home/...` | 本地路径 |
| `command` | `curl`, `git`, `docker`, `node` | PATH 命令 |
| `docker` | `docker run`, `镜像` | Docker 相关 |
| `次数` | `[3|3次|3轮|三次|三轮]` | 循环次数 |
| `直达` | `[200|成功|可用]` | 目标状态 |
| `超时` | `[60秒|120秒|1分钟]` | 超时时间 |
| `收敛` | `[2次|连续2次]` | 收敛阈值 |

**示例**：
```
用户：测试 /root/app.sh --次数=3 直到成功
↓
抽取出：
- path: /root/app.sh
- 次数: 3
- 直达: 成功
```

---

### 3️⃣ 模糊检测（Ambiguity Detection）

| 缺失项 | 问题 | 说明 |
|--------|------|------|
| `路径/命令` | “请提供要测试的脚本或工具名” | 必填项 |
| `次数` | “需要测试几次？默认 1 次” | 可选 |
| `直达目标` | “希望达到什么状态？200/成功/可用？” | 可选 |
| `脚本类型` | “这是 Bash 脚本还是 Python 程序？” | Guess + Ask |

---

### 4️⃣ 智能追问策略（Smart Follow-up）

| 场景 | 问题 | 后续 |
|------|------|------|
| 路径缺失 | “请提供要测试的脚本或工具名” | 等待用户输入 |
| 参数模糊 | “次数是 3 次吗？（--次数=3）” | 用户确认 |
| 类型未知 | “这是 Bash 脚本 (.sh) 还是 Python (.py)？” | 用户选择 |
| Docker 镜像 | “Docker 镜像名是？” | 用户输入 |

## 🛠️ v2.2 NLP 规则（简化版正则）

### 意图识别
```
/.*[测试|测|检查|验证].*/  → intent = "test"
/.*[安装|配置].*/        → intent = "install"
/.*[修复|调试].*/        → intent = "fix"
```

### 路径抽取
```
/\/[^\s]+/                → path: "/usr/bin/curl"
/\.\//                    → path: "./my_script.sh"
/^[a-zA-Z0-9_]+\..+$/     → filename: "app.py"
```

### 数字抽取
```
/(\d+)[次|轮|遍]/         → 次数: 3
/(\d+)[秒|分钟]/          → 超时: 60
/(\d+)/                   → 数字（通用）
```

### 目标抽取
```
/[-=]?(200|404|500|成功|可用|退出|exit=0)/ →直达目标
```

## 🔄 v2.2 工作流（完整）

```
1️⃣ 用户输入（自然语言）
   "测试 /root/app.sh，测 3 次直到成功"

2️⃣ v2.2 意图识别
   → intent = "test"

3️⃣ v2.2 实体抽取
   → path = "/root/app.sh"
   → 次数 = 3
   → 直达 = "成功"
   → 其他 = 默认值

4️⃣ v2.2 智能追问（如缺失）
   （本例完整，无需追问）

5️⃣ v2.2 生成标准指令
   → "测试 /root/app.sh：--次数=3 --直达=成功"

6️⃣ v2.1 执行测试
   → 执行 3 轮测试
   → 生成报告 + 进化

7️⃣ 输出结果
   → Markdown 报告 + memory 记录
```
