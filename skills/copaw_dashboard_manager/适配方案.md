## 📊 项目详情

### OpenClaw-bot-review 是什么？

**官方描述：**
> A lightweight web dashboard for viewing all your OpenClaw Bots/Agents/Models/Sessions status at a glance.

**核心功能：**
1. 🤖 **机器人总览** - 卡片墙显示所有代理状态
2. 📊 **模型列表** - 查看所有配置的提供商和模型
3. 💬 **会话管理** - 浏览每个代理的会话
4. 📈 **统计分析** - Token 消耗和响应时间趋势
5. 🔧 **技能管理** - 查看已安装的技能
6. ⚠️ **告警中心** - 配置告警规则
7. 🏢 **像素办公室** - 动画像素艺术办公室（特色功能！）
8. 🌐 **网关健康** - 实时网关状态

### 技术栈

- **前端：** Next.js + TypeScript
- **样式：** Tailwind CSS
- **数据源：** 直接读取 `~/.openclaw/openclaw.json`（无需数据库）
- **要求：** Node.js 18+

## 🔧 适配方案

### 方案 A: 并行运行（推荐）⭐⭐⭐⭐⭐

**思路：** OpenClaw-bot-review 作为独立服务运行，CoPaw 负责管理和访问。

```
┌─────────────────────────────────────┐
│         CoPaw 系统                   │
│  ┌─────────────────────────────┐    │
│  │  CoPaw Agent (如意)         │    │
│  │  - 对话交互                  │    │
│  │  - 技能执行                  │    │
│  └─────────────────────────────┘    │
│              ↓                       │
│  ┌─────────────────────────────┐    │
│  │  dashboard_manager 技能     │    │
│  │  - 启动/停止 Dashboard       │    │
│  │  - 检查状态                  │    │
│  │  - 查看统计                  │    │
│  └─────────────────────────────┘    │
└──────────────┬──────────────────────┘
               ↓
┌──────────────┴──────────────────────┐
│   OpenClaw-bot-review               │
│   (运行在 localhost:3000)           │
│   - 机器人监控 Dashboard             │
│   - 像素办公室                       │
│   - 统计分析                         │
└─────────────────────────────────────┘
```

**优点：**
- ✅ 实现简单（30 分钟）
- ✅ 互不干扰
- ✅ 各自独立升级
- ✅ CoPaw 可管理 Dashboard

**缺点：**
- ⚠️ 两个独立系统
- ⚠️ 需要分别配置

---

### 方案 B: 深度集成（高级）⭐⭐⭐⭐

**思路：** 将 Dashboard 功能封装为 CoPaw 技能，直接在 CoPaw 中访问。

**实现方式：**
1. 创建 CoPaw 技能调用 Dashboard API
2. 在 CoPaw 对话中显示统计信息
3. 将像素办公室嵌入 CoPaw 界面

**优点：**
- ✅ 统一体验
- ✅ 单一入口
- ✅ 数据共享

**缺点：**
- ⚠️ 实现复杂（2-3 小时）
- ⚠️ 需要修改 Dashboard 代码
- ⚠️ 维护成本高

---

### 方案 C: 桥接模式（折中）⭐⭐⭐⭐

**思路：** 创建桥接服务，让 CoPaw 和 Dashboard 可以互相访问对方功能。

**实现方式：**
1. CoPaw → Dashboard: 通过 HTTP API 控制
2. Dashboard → CoPaw: 嵌入 CoPaw 聊天组件
3. 共享配置和记忆

**优点：**
- ✅ 双向集成
- ✅ 灵活性高
- ✅ 实现难度中等

**缺点：**
- ⚠️ 需要桥接服务
- ⚠️ 增加系统复杂度

## 🚀 推荐方案 A 实施步骤

### 步骤 1: 安装 OpenClaw-bot-review

```bash
# 进入 CoPaw 目录
cd /root/.copaw

# 克隆项目
git clone https://github.com/xmanrui/OpenClaw-bot-review.git

# 进入目录
cd OpenClaw-bot-review

# 安装依赖
npm install
```

**预计时间：** 5-10 分钟

---

### 步骤 2: 配置连接

**前提：** 需要先安装 OpenClaw

```bash
# 安装 OpenClaw（如果还没有）
npm install -g openclaw

# 初始化 OpenClaw
openclaw onboard

# 配置完成后，Dashboard 会自动读取配置
```

**配置文件位置：** `~/.openclaw/openclaw.json`

---

### 步骤 3: 创建 CoPaw 管理技能

**位置：** `/root/.copaw/active_skills/dashboard_manager/`

**SKILL.md 内容：**

```markdown
# Dashboard Manager - CoPaw 技能

## 功能

- 启动/停止 Dashboard
- 检查运行状态
- 查看统计信息
- 管理配置

## 使用示例

用户：启动 Dashboard
助手：正在启动 OpenClaw-bot-review...
      ✅ 启动完成！
      🌐 访问：http://localhost:3000

用户：Dashboard 状态
助手：✅ 运行中
      📊 端口：3000
      ⏱️ 运行：2 小时
```

---

### 步骤 4: 启动 Dashboard

**方式 1: 手动启动**

```bash
cd /root/.copaw/OpenClaw-bot-review
npm run dev
```

**方式 2: 通过 CoPaw 启动**

```
你：启动 Dashboard
CoPaw：正在启动 OpenClaw-bot-review...
      ✅ 启动完成！
      🌐 访问：http://localhost:3000
```

**方式 3: 系统服务**

```bash
# 创建 systemd 服务
sudo nano /etc/systemd/system/openclaw-dashboard.service

# 配置内容：
[Unit]
Description=OpenClaw Dashboard
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/.copaw/OpenClaw-bot-review
ExecStart=/usr/bin/npm run dev
Restart=always

[Install]
WantedBy=multi-user.target

# 启用服务
sudo systemctl enable openclaw-dashboard
sudo systemctl start openclaw-dashboard
```

## 🔍 故障排除

### 问题 1: Dashboard 无法启动

**检查：**
```bash
# 检查 Node.js 版本
node --version  # 需要 18+

# 检查端口占用
lsof -i :3000

# 检查依赖
cd /root/.copaw/OpenClaw-bot-review
npm install --force
```

**解决：**
```bash
# 升级 Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs
```

---

### 问题 2: 无法连接 OpenClaw

**检查：**
```bash
# 检查 OpenClaw 配置
cat ~/.openclaw/openclaw.json

# 检查 Gateway 是否运行
curl http://localhost:18789/health
```

**解决：**
```bash
# 启动 OpenClaw Gateway
openclaw gateway run
```

---

### 问题 3: 像素办公室不显示

**检查：**
```bash
# 查看浏览器控制台错误
# 检查是否加载了 pixel-office 组件
```

**解决：**
```bash
# 清除缓存
rm -rf .next
npm run dev
```
