### 🎯 强制公开政策（永久生效）

> **规则：以后所有新建的 GitHub/Gitee 仓库必须为 public（公开），禁止私有！**

#### ✅ 原因

- 开源项目需要透明协作
- 客户/用户需要自由访问
- 避免误建私有库
- 与如意核心理念一致（共享、透明、善良）

#### 📋 执行标准

| 平台 | 创建命令 | 参数说明 |
|------|---------|----------|
| GitHub | `curl -X POST ... -d "{"private":false}"` | 明确设为公开 |
| Gitee | `curl -X POST ... -d "visibility=0"` | 但 **受用户默认设置影响**，可能仍为私有 |

#### ⚠️ Gitee 注意事项

- Gitee API 创建时 `private=false` 或 `visibility=0` **可能无效**
- Gitee API **无法修改**公开/私有权限（安全限制）
- ✅ 解决方案：
  1. Web 页面手动修改为公开（https://gitee.com/用户名/仓库/settings）
  2. 或提交工单请求客服协助

#### 📁 新建脚本

- 创建文件：`bin/create_public_repo.sh`
- 功能：一键创建公开仓库（支持 `github` / `gitee`）
- 权限：已授予 `chmod +x`
