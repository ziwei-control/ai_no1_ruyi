## ✅ 自动触发配置

### 触发模式

技能已配置为**自动检测并触发**，无需手动调用！

#### 自动识别以下内容：

1. **URL 链接** - 任何包含 `http://` 或 `https://` 的消息
   ```
   用户：https://example.com/article
   ```

2. **提取指令 + URL** - 包含"读取"、"获取"、"提取"等关键词
   ```
   用户：帮我读取这篇文章 https://...
   用户：获取网页内容 https://...
   ```

3. **特定域名优化** - 针对以下域名自动优化策略：
   - `mp.weixin.qq.com` - 微信公众号
   - `zhuanlan.zhihu.com` - 知乎专栏
   - `juejin.cn` - 掘金
   - `github.com` - GitHub 项目

## 💡 核心功能

### 1. 多策略智能提取

| 方法 | 优先级 | 速度 | 成功率 |
|------|--------|------|--------|
| r.jina.ai API | 1️⃣ | ⚡⚡⚡ | 95%+ |
| curl 移动端 | 2️⃣ | ⚡⚡ | 80%+ |
| browser_use | 3️⃣ | ⚡ | 70%+ |
| GitHub 搜索 | 4️⃣ | ⚡⚡ | 60%+ |
| 缓存服务 | 5️⃣ | ⚡⚡ | 50%+ |

**智能 Fallback：** 自动检测失败并切换到下一方法

### 2. 自动存档系统

所有提取的内容自动保存到：
```
/root/.copaw/memory/web_archive_<domain>_<timestamp>.md
```

**档案包含：**
- ✅ 元数据（URL、时间、方法、状态）
- ✅ 提取的正文内容
- ✅ 技术信息（尝试次数、耗时）
- ✅ 建议和指导

### 3. 特殊网站优化

- **微信公众号** - 移动端 UA + GitHub 补充
- **知乎专栏** - 浏览器渲染
- **GitHub** - 直接提取 README
- **技术博客** - 优先 Markdown 提取

## 🔧 配置选项

### 查看当前配置

```bash
cat /root/.copaw/active_skills/web_content_extractor/SKILL.md
```

### 修改超时时间

编辑 `/root/.copaw/skills/web_content_extractor/extract.sh`：

```bash
TIMEOUT_LONG=90    # r.jina.ai 超时（原 60 秒）
TIMEOUT_XLONG=120  # curl 超时（原 90 秒）
```

### 禁用自动触发

编辑 `/root/.copaw/active_skills/web_content_extractor/SKILL.md`：

```yaml
auto_trigger:
  enabled: false  # 改为 false 禁用自动触发
```

### 修改存档目录

编辑 `extract.sh`：

```bash
MEMORY_DIR="/your/custom/path"
```

## 🔍 故障排除

### 问题 1: 技能没有自动触发

**检查：**
1. 消息中是否包含 URL（http://或 https://）
2. 检查配置：`cat /root/.copaw/active_skills/web_content_extractor/SKILL.md`
3. 查看系统日志

**测试：**
```bash
# 手动测试脚本
cd /root/.copaw/skills/web_content_extractor
./extract.sh "https://example.com"
```

### 问题 2: 提取失败

**解决：**
1. 检查网络连接
2. 手动访问 URL 确认网站正常
3. 稍后重试（可能临时故障）
4. 查看存档文件中的详细错误信息

### 问题 3: 内容不完整

**原因：** 网站有强反爬虫

**解决：**
1. 技能会自动尝试多种方法
2. 如仍失败，建议手动复制原文
3. 查看已获取的部分内容
