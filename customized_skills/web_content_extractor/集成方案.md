# Web Content Extractor - CoPaw 集成方案

## 📊 现状分析

### CoPaw 的 Skills 机制

CoPaw 系统中的 `active_skills` 目录用于存放**已激活的技能**，系统会：

1. ✅ **自动加载** - 启动时加载 `active_skills/` 下的所有技能
2. ✅ **提供给 Agent** - Agent 可以看到并选择使用这些技能
3. ❌ **不支持自动触发** - 没有基于消息模式（如 URL）的自动触发机制

### 当前实现

我们已将 `web_content_extractor` 安装到：
```
/root/.copaw/active_skills/web_content_extractor/
```

Agent **可以访问**这个技能，但需要：
- ❌ 不会自动检测 URL 并触发
- ✅ 需要用户明确调用或 Agent 主动选择使用

---

## 🎯 解决方案

### 方案 1: Agent 系统提示增强（推荐）

通过在系统提示中添加规则，让 Agent 自动检测 URL 并调用技能。

**实现方式：**

在对话开始时或通过配置，添加以下系统提示：

```markdown
## 自动网页内容提取规则

当用户消息中包含 URL（http://或 https://）时，你必须：

1. **立即识别** - 检测到任何 http://或 https://链接
2. **自动调用** - 使用 web_content_extractor 技能提取内容
3. **显示进度** - 向用户展示提取进度
4. **保存存档** - 内容自动保存到 memory/目录
5. **返回结果** - 展示提取的内容摘要

示例：
用户：https://example.com/article
助手：🔄 正在提取网页内容... [调用 web_content_extractor]
```

**优点：**
- ✅ 无需修改 CoPaw 核心代码
- ✅ 立即生效
- ✅ 简单可靠

**缺点：**
- ⚠️ 依赖 Agent 遵守规则

---

### 方案 2: 消息预处理 Hook

在消息到达 Agent 之前，预处理并插入技能调用指令。

**实现方式：**

创建预处理脚本：

```python
# /root/.copaw/hooks/url_detector.py
import re
from copaw.agents.hooks import Hook

class URLDetectorHook(Hook):
    """检测 URL 并自动触发 web_content_extractor 技能"""
    
    URL_PATTERN = re.compile(r'https?://[^\s]+')
    
    def on_user_message(self, message: str, context: dict) -> str:
        # 检测 URL
        urls = self.URL_PATTERN.findall(message)
        if urls:
            # 插入技能调用指令
            instruction = f"\n\n[SYSTEM: 检测到 URL，请调用 web_content_extractor 提取以下内容：{', '.join(urls)}]"
            return message + instruction
        return message
```

**注册 Hook：**

在 CoPaw 配置中注册此 Hook。

**优点：**
- ✅ 真正的自动触发
- ✅ 不依赖 Agent 判断

**缺点：**
- ⚠️ 需要修改 CoPaw 代码或配置
- ⚠️ 需要重启服务

---

### 方案 3: 增强型 Skill（当前最可行）

修改 `web_content_extractor` 技能，让它在 SKILL.md 中明确说明使用场景，并依赖 Agent 的智能选择。

**实现方式：**

在 `SKILL.md` 的开头添加醒目的使用说明：

```markdown
## ⚠️ 重要：何时使用此技能

**当用户消息中包含 URL 时，必须使用此技能！**

触发条件：
- 消息包含 http://或 https://
- 用户说"读取"、"获取"、"打开"等词 + URL
- 用户提供任何网页链接

使用示例：
用户：https://example.com
助手：[立即调用 web_content_extractor]
```

**优点：**
- ✅ 利用 Agent 的智能判断
- ✅ 无需修改系统
- ✅ 符合 CoPaw 设计理念

**缺点：**
- ⚠️ 依赖 Agent 的判断力
