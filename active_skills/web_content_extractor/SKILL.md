# Web Content Extractor - 网页内容自动提取

此 skill 用于智能提取网页正文内容，支持多种提取策略和自动 fallback 机制。

## 自动触发规则

### 触发条件

当用户消息中包含以下任一模式时，**自动触发**此技能：

1. **URL 链接** - 包含 `http://` 或 `https://`
2. **提取指令** - 包含"读取"、"获取"、"提取"、"打开"、"查看"、"保存"、"抓取"等关键词
3. **特定域名** - 微信公众号、知乎、掘金、GitHub 等

### 触发示例

```
用户：帮我读取这篇文章 https://example.com/article
用户：获取这个网页的内容 https://mp.weixin.qq.com/s/xxx
用户：https://github.com/openclaw/openclaw 看看这个项目
用户：提取这篇文章的内容：https://zhuanlan.zhihu.com/p/123
用户：保存这个网页到本地
```

## 核心功能

### 1. 多策略提取引擎

按优先级自动尝试以下方法：

| 方法 | 优先级 | 速度 | 成功率 | 适用场景 |
|------|--------|------|--------|----------|
| **r.jina.ai API** | 1️⃣ | ⚡⚡⚡ | ⭐⭐⭐⭐⭐ | 普通网页、博客、文档 |
| **curl 移动端模拟** | 2️⃣ | ⚡⚡ | ⭐⭐⭐⭐ | 微信公众号、知乎 |
| **browser_use 浏览器** | 3️⃣ | ⚡ | ⭐⭐⭐ | 动态 JS 内容 |
| **GitHub 搜索补充** | 4️⃣ | ⚡⚡ | ⭐⭐⭐ | 技术文章相关项目 |
| **缓存服务备用** | 5️⃣ | ⚡⚡ | ⭐⭐ | Archive/Google 缓存 |

### 2. 智能 Fallback 机制

```
开始提取
   ↓
尝试 r.jina.ai (45 秒超时)
   ├─ 成功 → 提取 Markdown → 保存 ✅
   └─ 失败/超时 → 下一步
   ↓
尝试 curl 移动端 (90 秒超时)
   ├─ 成功 → 解析 HTML → 保存 ✅
   └─ 失败/超时 → 下一步
   ↓
尝试 browser_use (60 秒超时)
   ├─ 成功 → 截图 + 快照 → 保存 ✅
   └─ 失败/超时 → 下一步
   ↓
尝试 GitHub 搜索 (如有关键词)
   ├─ 成功 → 获取相关项目 → 保存 ✅
   └─ 失败 → 下一步
   ↓
尝试缓存服务
   ├─ 成功 → 提取内容 → 保存 ✅
   └─ 失败 → 报告错误 ❌
```

## 执行流程

### 步骤 1: 识别 URL 和关键词

从用户消息中提取：
- **URL 链接** - 使用正则提取所有 http/https 链接
- **关键词** - 从上下文提取用于 GitHub 搜索的关键词
- **特殊指示** - 如"保存为 PDF"、"只要正文"等

### 步骤 2: 显示提取进度

向用户展示实时进度：

```
🔄 正在提取网页内容...
URL: https://...

✓ 方法 1: r.jina.ai API - 提取中...
```

### 步骤 3: 执行提取策略

#### 方法 1: r.jina.ai API（优先）

```bash
cd /root/.copaw/skills/web_content_extractor
./extract.sh "<URL>"
```

**成功标准：**
- 返回内容 > 100 字符
- 不包含"环境异常"、"CAPTCHA"、"404"等错误关键词
- 包含实际内容（Title + 正文）

**失败处理：**
- 立即记录失败原因
- 无缝切换到方法 2

#### 方法 2: curl 移动端模拟

如果 r.jina.ai 失败，脚本会自动尝试 curl 移动端。

#### 方法 3: browser_use 浏览器

如果前两种方法都失败，使用浏览器：

```json
{"action": "open", "url": "<URL>"}
{"action": "snapshot"}
{"action": "screenshot", "full_page": true, "path": "web_screenshot.png"}
```

**适用场景：**
- 动态加载内容（JavaScript）
- 需要截图的页面
- 前两种方法都失败时

**注意：**
- 设置 60 秒超时
- 自动检测验证码/登录页面
- 遇到验证时暂停并提示用户

#### 方法 4: GitHub 搜索补充

如果提取到技术相关关键词：

```bash
curl -s --max-time 60 \
  "https://r.jina.ai/http://github.com/search?q=<关键词>"
```

**用途：**
- 获取相关开源项目
- 补充技术文档
- 提供替代信息源

### 步骤 4: 生成存档文件

脚本自动保存到：
```
/root/.copaw/memory/web_archive_<domain>_<timestamp>.md
```

### 步骤 5: 返回结果

向用户展示：

## 特殊情况处理

### 微信公众号文章

**特点：**
- 强反爬虫机制
- 需要移动端 UA
- 经常触发验证码

**策略：**
1. 优先使用 curl 移动端模拟
2. 提取关键词搜索 GitHub 相关项目
3. 整合多个信息源
4. 明确告知用户限制

**话术示例：**
```
⚠️ 微信公众号文章触发反爬虫机制

✅ 已获取：
- 文章标题
- 部分 HTML 内容
- 相关 GitHub 项目（6 个）

❌ 无法获取：
- 完整正文（需要人工验证）

💡 建议：
1. 您手动打开链接复制原文给我
2. 或提供截图我来分析
3. 我已保存已获取的内容到档案
```

### 知乎专栏

**特点：**
- 需要登录才能看全文
- 动态加载内容

**策略：**
1. 尝试 r.jina.ai（有时可绕过）
2. 使用 browser_use 渲染 JS
3. 如遇登录墙，提示用户

### GitHub 项目

**特点：**
- 公开 API 友好
- Markdown 格式

**策略：**
1. 直接使用 r.jina.ai
2. 提取 README.md
3. 获取项目统计信息（stars、forks 等）

### 需要登录的网站

**处理流程：**
1. 检测到登录页面
2. 暂停自动操作
3. 提示用户手动登录
4. 用户确认后继续

**话术：**
```
🔐 检测到需要登录

请先在浏览器中完成登录：
1. 打开链接：https://...
2. 登录账号
3. 回复"继续"或"登录好了"

我将从登录后的页面继续提取内容。
```

## 性能优化

### 超时设置

| 方法 | 超时时间 | 理由 |
|------|----------|------|
| r.jina.ai | 45 秒 | 通常 2-5 秒完成 |
| curl 移动端 | 90 秒 | 可能需要多次重试 |
| browser_use | 60 秒 | 浏览器启动较慢 |
| GitHub 搜索 | 45 秒 | API 响应较快 |
| 缓存服务 | 30 秒 | 快速失败 |

### 并发策略

- **串行执行** - 按优先级依次尝试
- **快速失败** - 检测到错误立即切换
- **结果缓存** - 相同 URL 不重复提取（1 小时内）

## 安全和伦理

### 使用准则

- ✅ 仅用于个人学习和研究

## 故障排除

### 常见问题

**Q: 所有方法都失败？**
A: 
1. 检查网络连接
2. 手动访问 URL 确认网站正常
3. 稍后重试（可能临时故障）
4. 建议用户手动复制内容

**Q: 提取的内容不完整？**
A:
1. 检查是否触发反爬虫
2. 尝试使用 browser_use 渲染
3. 搜索 GitHub 获取替代信息
4. 建议用户提供原文

**Q: 中文乱码？**
A:
1. 网站可能使用 GBK 编码
2. 使用 iconv 转换
3. 建议手动访问复制

**Q: 脚本执行卡住？**
A:
1. 检查超时设置是否合理
2. 使用 timeout 命令包裹
3. 查看日志输出定位问题
4. 重启浏览器进程
