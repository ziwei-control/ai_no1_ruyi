# 🤖 智能一号 如意（AI No.1 如意）· 企业级快速上手指南

## 🎯 一句话说明

**把用户说的自然语言，变成生产级测试计划 → 自动执行 → 生成报告 → 自动修复**

---

## 🚀 企业级使用方式

### 方式 1：自然语言输入（v2.2）

```
用户：你好，能帮我测试 /root/.copaw/scripts/app.sh 吗？
测 3 次，直到成功，超时 120 秒
```

→ 如意 自动调度：
- v2.2 NLP层 → 提取参数
- v2.1 路径层 → 解析路径 + Shebang
- v2.0 循环层 → 生成用例 + 执行
- 报告层 → 生成 Markdown
- 进化层 → 失败 → 修复版 Skill

---

### 方式 2：半结构化输入（最常用）

```
测试 /root/app.sh：
  --次数=3
  --直达=成功
  --超时=120
  --收敛=2
```

→ 直接调用 v2.1/v2.0 执行

---

### 方式 3：结构化指令（快速执行）

```
测试 curl：--次数=2 --直达=200
测试 docker run nginx:latest：--次数=3 --直达=启动成功
```

→ 最快路径，适合熟练用户

## 📋 如意 标准指令格式

```
测试 <目标>：
  --次数=N              # 最多 N 轮（默认 1）
  --直达=目标           # 达到目标即停（200/成功/可用）
  --超时=N              # 单次超时 N 秒（默认 60）
  --收敛=N              # 连续 N 次相同结果则停（默认 0）
  --verbose             # 详细模式（每轮输出）
  --skip-confirm        # 跳过追问（直接执行）
```

## 🧠 典型企业场景

### 场景 1：本地脚本测试（推荐）

```
测试 /root/.copaw/scripts/demo.sh：
  --次数=3
  --直达=0
  --超时=120
  --verbose
```

→ 如意 自动：
1. 检查脚本存在性 + 权限
2. 读取 Shebang（`#!/bin/bash`）
3. 生成 Bash 用例（参数/边界/异常）
4. 循环执行 3 轮（超时 120s）
5. 检查收敛（如 `--收敛=2`）
6. 生成 report_如意_demo_*.md

---

### 场景 2：Docker 镜像测试

```
测试 docker run nginx:latest：
  --次数=2
  --直达=200
  --超时=30
```

→ 如意 自动：
1. 检查镜像是否存在
2. 生成 Docker 用例（启动/健康检查/退出）
3. Docker 镜像测试（带 timeout）
4. 报告 + 进化

---

### 场景 3：Python 程序测试

```
测试 ./my_app.py：
  --次数=5
  --直达=成功
  --超时=120
  --收敛=2
```

→ 如意 自动：
1. 读取 Shebang（`#!/usr/bin/env python3`）
2. 生成 Python 用例（参数/异常/边界）
3. 循环执行 + 收敛检查
4. 报告 + 进化

## 🎯 如意 报告结构

| 字段 | 说明 |
|------|------|
| `Header` | NLP 原始输入 + 时间戳 |
| `Entities` | 提取的参数（次数/直达/超时/收敛） |
| `Origin` | 原始路径 |
| `Parsed Name` | 解析后名称 |
| `Script Type` | Bash/Python/Shell/Docker |
| `Shebang` | 脚本第一行 |
| `Test Cases` | 用例列表 + 输出 |
| `Time Line` | 多轮时间线（失败/成功/收敛） |
| `Diagnostic` | 错误分类（env/args/logic） |
| `Suggestion` | 修复建议 + 新 Skill |
